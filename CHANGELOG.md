# Changelog

## v3.5.0 (2026-07-17)

### 新功能

- **剪贴板多选合并：** 可从底栏进入多选模式，按列表顺序选择并合并 2 至 20 个同主分类条目；文本类内容合并为新的剪贴板条目，图片合并为可继续编辑和排序的备忘录，原条目保持不变
- **批量删除：** 多选模式允许混合选择任意分类条目，并可一次删除所选内容；开启回收站时批量移入回收站，关闭时经确认后永久删除
- **实验多选模式：** 实验功能新增默认关闭的“多选模式”开关；开启后显示“选择条目”入口，并支持 `Ctrl+点击` 直接开始多选及按 `Delete` 调用批量删除确认
- **底栏视觉对齐：** 经典与新版 UI 的状态字段、清除历史、选择及批量操作统一为 24px 高度和一致行高，并校正带文字按钮中 Lucide 图标与中文字体的视觉中心
- **底栏总数修复：** 底栏条目数改用当前模块或分类的数据库总数，不再被剪贴板、备忘录及回收站每页 50 条的增量加载上限截断
- **合并后处理：** 每次合并前通过软件样式弹窗选择“仅合并”或“合并并删除原条目”，弹窗会明确说明原条目将进入回收站还是被永久删除
- **合并结果去重：** 文本合并结果复用现有内容哈希去重，数据库中已有相同内容时直接定位已有条目，不重复写入
- **多标签分类模式：** 只有开启实验功能中的“剪贴板多标签显示”后，辅助标签才参与 Tab 分类和数量统计；关闭时列表和计数统一只按主标签归类
- **Windows 路径识别修复：** 新收录内容支持 `E:/folder/file.ext` 形式的正斜杠盘符路径，避免被误判为代码和文本；历史条目不会自动改写，可在实验功能中确认后手动重新识别当前存储
- **历史分类手动更新：** 实验功能新增“更新历史分类”操作，悬浮提示建议在版本或分类规则更新后使用；执行前通过软件样式弹窗确认，仅重算主标签、辅助标签和搜索索引
- **分类清除范围：** 在分类标签中清除历史时，弹窗可选择仅清除当前标签或清除全部未置顶记录；当前标签范围与多标签显示开关的分类语义保持一致
- **清除去向提示：** “清除历史”确认弹窗会按回收站开关明确说明条目将进入回收站并可恢复，或被永久删除且无法恢复
- **回收站快速清空：** 回收站底栏新增“清空”按钮，按当前剪贴板或备忘录子标签批量永久删除全部记录；执行前明确提示数量且不可恢复，本地与外部存储均使用单次批量操作
- **永久删除语义统一：** 外部 PostgreSQL 的永久删除、批量删除、过期清理和回收站清空改为物理删除，不再只写入隐藏标记；普通“清除历史”也不会误删已经进入回收站的记录
- **清空状态一致性：** 清空回收站后立即废弃尚未完成的旧列表与分页请求，避免慢速外部请求短暂把已删除条目重新显示出来
- **更新检查限流回退：** GitHub Releases API 返回限流或临时错误时，自动通过公开的 Latest Release 重定向继续判断版本，避免自动与手动检查直接失效
- **置顶快捷键切换：** 窗口置顶且仍可见时，按一次全局快捷键即可隐藏；普通模式下窗口被其他应用遮挡时仍先拉到前台，最小化窗口仍正常恢复
- **条目列表绘制修复：** 经典界面的剪贴板、备忘录和回收站改由列表背景与固定间距形成分隔线，并移除可能触发 WebView2 合成层漏绘的条目绘制隔离；列表条目固定为不可收缩，避免新版 UI 在大量条目同时加载时被压缩成横线
- **经典界面留白：** 条目不足一屏时，列表剩余区域恢复为主背景色；分隔线仍只在实际条目之间绘制
- **设置菜单分组：** 系统设置与功能设置统一采用实验功能菜单的轻量标题与右侧细线，减少面板中的重复边框
- **图片合并标签修复：** 合并图片生成备忘录时，图片 Base64 数据不再参与文本分类，避免误显示“代码”自动标签；旧的纯图片备忘录也会在界面中隐藏该历史误判标签

### 技术细节

- 本地 SQLite 与外部 PostgreSQL 统一增加按 ID 批量读取接口，合并操作不再逐条请求远程数据库
- 本地 SQLite 使用单事务批量删除，外部 PostgreSQL 使用单次 `ANY($1)` 更新，避免远程模式逐条执行网络请求
- 合并规则、选择限制与结果生成提取为独立前端模块，并补充分类一致性、列表顺序、数量边界和图片备忘录生成测试
- 多标签分类模式随设置切换即时刷新当前列表和 Tab 数量，本地 SQLite 与外部 PostgreSQL 保持相同查询语义
- SQLite schema `9` 保留手动重分类能力；PostgreSQL schema `9` 清理旧软删除记录、改用物理删除，并为当前数据库保存已应用的分类规则版本。手动重分类仍不会在启动或连接数据库时自动执行
- 实验功能会读取当前存储的分类规则状态，区分“历史数据已更新”和“建议重新识别”，不同外部数据库独立记录
- 剪贴板列表与回收站备忘录的分页、旧请求失效和增量加载抽离为独立 Hook；SQLite 与 PostgreSQL 建表升级分别迁入专用迁移模块
- 设置、存储、实验功能面板及备忘录模块拆分为动态构建块，主入口 JavaScript 从约 `533 KB` 降至约 `244 KB`
- 新增 GitHub Actions CI，在临时 PostgreSQL 16 服务上执行远端 schema、读写、搜索、冲突和删除烟测，同时运行前端 lint/测试/构建与 Rust fmt/测试/Clippy
- 条目取消 `content-visibility` 与 paint containment，分隔色由列表容器统一绘制；剪贴板和备忘录条目显式禁止 flex 收缩，保证滚动列表中的实际内容高度
- PostgreSQL 实时刷新保留 `LISTEN / NOTIFY`，移除从未读取且会持续增长的 `sync_events` 表，减少每次远端写入的额外放大
- 移除未使用的 Tauri Shell 前后端依赖与权限；完整保留 Tauri 平台图标集，便于后续适配 macOS、移动端或 Microsoft Store
- 按入口导入图、CSS 引用、Rust 模块和依赖重新审计迭代残留；未发现可安全删除的失联业务文件，恢复并保留完整平台图标模板
- 回收站子标签的重复内联样式收敛到统一 CSS 类，减少主应用组件中的样式分支
- 应用、Tauri 配置、Rust 包、双语文档和官网软件版本同步到 `3.5.0`

## v3.4.0 (2026-07-16)

### 改进

- **远程连接自恢复：** 外部 PostgreSQL 连接池改为按需建连并缩短空闲连接生命周期；休眠或网络抖动后若取连接失败，会安全废弃对应旧连接池并重建重试一次
- **远程图片请求调度：** 图片 Base64 预览改为单通道排队并合并同一条目的重复请求，避免图片列表占满远程任务和数据库连接，影响保存、搜索与计数刷新
- **备忘录标签去重：** 手动标签与自动标签改为独立存储，自动标签只按当前界面语言显示一次；本地与外部数据库会清理历史记录中重复积累的中英文标签
- **新建备忘录位置：** 正在新建的编辑框固定显示在新建按钮下方，不再被置顶条目挤到列表中间；保存或取消后恢复正常的置顶排序
- **图片搜索索引：** 本地 SQLite 与外部 PostgreSQL 新增独立可读文本索引，剪贴板图片和备忘录内嵌图片的 Base64 数据不再参与搜索，文字、标签、分类和图片尺寸仍可正常匹配
- **历史数据迁移：** 本地与外部数据库会为已有条目回填搜索文本；PostgreSQL 使用数据库端批量迁移，并按外部连接记录已完成的 schema 版本，避免重复迁移拖慢启动
- **远端启动顺序：** 外部模式升级时先完成远端表结构检查，再启动剪贴板监听和实时通知，避免新字段迁移前发生写入失败
- **确认弹窗层级：** 已保存连接的删除确认弹窗统一挂载到应用根层，确保新版界面的背景模糊效果与清除条目弹窗一致
- **Devpost 项目说明：** 双语 README 新增 Codex 与 GPT-5.6 使用说明，明确人类主导范围、协作环节和运行时隐私边界
- **编辑内容去重：** 剪贴板条目在编辑后会同时保存当前内容哈希；再次复制编辑后的内容不会被重新收录。本地 SQLite 与外部 PostgreSQL 都会迁移已有数据并建立查询索引
- **图片操作完善：** 图片条目悬浮操作区新增应用内预览和 PNG 另存为入口；预览使用软件统一样式弹窗，支持点击遮罩或 `Esc` 关闭
- **更新提示可读性：** 自动更新弹窗中的下载按钮补充独立悬浮、按下和焦点样式，确保强调色下的文字对比稳定
- **备忘录时间与空标题：** 统一旧 SQLite 时间戳为 UTC 格式，避免切换页面后“编辑于”时间跳变；空标题按当前语言显示为“无标题”或“Untitled”
- **拖放收录：** 可将文字、链接、图片或图片文件直接拖入主窗口收录；拖放内容会同步写入系统剪贴板，并复用本地/外部存储、分类和去重流程，备忘录编辑区仍保留原有的图片插入行为
- **编辑历史收敛：** 保存未修改的剪贴板条目不会产生重复内容或“编辑于”记录；后续改回首次复制的原文时会自动清理原文折叠区和编辑时间，编辑时间与“查看原文”统一显示在条目右下角

### 技术细节

- 修正备忘录标签与已保存连接选中态混用边框简写和 `borderColor` 时产生的 React 重渲染警告
- 新增远程图片内容加载队列测试，覆盖串行执行、同条目请求去重和失败后继续调度
- 新增搜索文本提取与数据库迁移回归测试，并使用真实 PostgreSQL 验证普通文本、图片描述和 Base64 排除逻辑
- 本地 schema 升级到 `8`、外部 PostgreSQL schema 升级到 `6`，为编辑后内容去重补充可迁移的当前内容哈希
- 应用、Tauri 配置、Rust 包和双语文档版本同步到 `3.4.0`
- Windows 下改由前端 HTML5 拖放处理主窗口导入，支持 PNG、JPEG、GIF 与 WebP 图片并限制单次导入体积
- 新增剪贴板无改动保存和恢复原文的本地/外部存储回归测试，确保编辑元数据与内容状态一致
- 清理本地代理工具生成目录，并补充 `.agents/` 忽略规则

## v3.3.0 (2026-07-15)

### 改进

- **备忘录加载性能**：自动标签在创建和编辑时直接持久化，本地与外部数据库各执行一次历史数据迁移，列表读取不再额外调用标签识别命令
- **大列表加载**：剪贴板、备忘录及回收站改为每页 50 条增量加载，并补充常用排序条件的本地 SQLite 与外部 PostgreSQL 索引
- **设置写入稳定性**：设置修改按顺序持久化，失败时只回滚当前字段，避免快速切换多个开关时覆盖其他已修改状态
- **更新检查可靠性**：更新检查拆分为独立模块，使用标准语义化版本比较，增加请求超时和 HTTP 状态校验，并优先返回 Windows 安装器直链
- **菜单事件复用**：设置、存储和实验功能菜单统一使用外部点击 Hook，减少重复的全局事件监听代码
- **连接列表一致性**：已保存数据库连接仅在写入成功后更新界面状态，避免持久化失败时界面与配置不一致
- **语言切换稳定性**：国际化上下文独立为稳定模块，语言写入按顺序执行并在失败时回滚，修复切换外部数据库后界面变英文且语言按钮失效的问题
- **存储切换刷新**：本地与外部存储切换只触发一次数据刷新，避免远端图片较多时重复请求持续覆盖加载状态；休眠唤醒仍保留分阶段重试
- **远端图片按需加载**：外部存储列表不再一次传输全部图片 Base64，条目先显示并在图片接近可视区时按需加载，避免大数据量下计数已更新但列表因超时显示为空
- **存储失败回滚**：保存配置或切换已保存连接失败时同步恢复表单、连接指示和已生效配置，并阻止重复提交
- **备份界面整理**：移除“备份/恢复”的测试版标签，继续保留跨版本恢复兼容性提示
- **菜单样式边界**：经典 UI 的设置类菜单恢复实体背景；新版 UI 将菜单移出标题栏合成层，并修正生产 CSS 对背景滤镜的错误裁剪，解决正式构建中毛玻璃失效和经典界面被连带改动的问题
- **菜单线条清晰度**：新版 UI 为设置、存储设置和实验功能菜单统一增加专用边框色，提高标题分隔线、分区线、输入框和按钮边框在毛玻璃背景上的辨识度

### 技术细节

- 新增 Vitest 与 Testing Library 前端测试基线，覆盖设置 Schema 加载、设置写入队列、语言切换失败回滚和菜单外部点击行为
- 新增更新检查、备忘录标签推断和标签迁移测试，覆盖 `3.10.0 > 3.9.0`、安装器选择与历史数据回填
- 后端更新检查拆分到 `update.rs`，剪贴板与备忘录命令拆分到 `commands/`；备忘录分页和拖拽交互分别收拢到独立 Hook
- 应用、Tauri 配置、Rust 包、双语文档和官网软件版本展示同步到 `3.3.0`

## v3.2.0 (2026-07-13)

### 改进

- **远程存储流畅性**：外部 PostgreSQL 改用最多 4 个连接的受控连接池，远程查询、搜索、计数和写入统一在后台阻塞任务中执行，避免网络 I/O 占用界面命令线程
- **设置加载性能**：应用设置和远程连接配置支持批量读取与事务写入，减少启动和打开设置面板时的重复 IPC 与数据库锁竞争
- **多设备实时刷新**：客户端接入 PostgreSQL `LISTEN / NOTIFY`，远端写入会通知其他设备；前端合并短时间内的通知后刷新当前列表与计数，并在断线后自动重连
- **并发编辑保护**：剪贴板条目和备忘录使用远端 `version` 乐观锁，其他设备已修改的内容不会被静默覆盖，界面会刷新最新内容并提示重新编辑

### 技术细节

- **设置状态统一**：新增设置 Schema、`useAppSettings`、共享 Provider 和通用设置行组件，移除 `App`、设置菜单和实验功能菜单之间的重复状态与开关代码
- **连接生命周期**：切换外部数据库配置时自动失效旧连接池；连接池限制并发连接数，并设置连接检测、空闲回收和最大生命周期
- **迁移管理**：远端 schema 使用 `schema_migrations` 记录版本，并用 PostgreSQL advisory lock 串行化初始化和后续迁移
- **存储边界**：本地 / 外部存储分发收拢到 `storage_backend`，已保存连接资料拆分为独立前端模块，减少命令层和设置面板的重复逻辑

## v3.1.0 (2026-07-10)

### 新功能

- **多数据库连接管理**：成功连接并切换过的外部 PostgreSQL 数据库会保存为本机连接配置，最多保留 12 条
- **快速切换数据库**：存储设置中新增“已保存连接”列表，可直接选择配置或重新测试后切换到对应数据库
- **连接删除确认**：删除已保存连接前使用软件统一样式弹窗确认，删除操作不会影响外部数据库及其中的数据
- **多色模式(Tab标签)（实验功能）**：可选地让文本、链接、图片、代码、邮箱和路径 Tab 的选中背景显示各自分类色，默认关闭

### 改进

- **连接记录去重**：按连接方式、地址、数据库、账号和 SSL 配置识别相同连接，重复使用时更新时间而不是生成重复记录
- **切换失败保护**：切换已保存连接失败时恢复此前有效配置，避免当前存储状态被无效连接覆盖
- **双语界面补全**：已保存连接、使用、删除和最后使用时间等字段支持中文与英文
- **编辑与原文复制**：点击条目复制最新编辑结果；展开后点击原文内容可复制首次捕获的原始内容，二者互不覆盖
- **新版 UI 交互**：分类 Tab 的选中底色改为平滑滑动；设置类菜单和开关切换减少视觉闪烁
- **界面细节修复**：修复旧版备忘录搜索框聚焦时出现深色底边的问题，并整理新版备忘录新建按钮样式

### 技术细节

- **版本同步**：应用、Tauri 配置、Rust 包、双语文档和官网软件版本展示同步到 `3.1.0`
- **质量验证**：前端构建、ESLint、Rust 格式检查、单元测试和 Clippy 检查均通过

## v3.0.0 (2026-07-09)
