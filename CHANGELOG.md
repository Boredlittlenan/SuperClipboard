# Changelog

## v2.20.0 (2026-07-08)

### 新功能

- **存储设置(测试版)**：系统设置中新增开关，开启后才在设置按钮旁显示存储配置入口
- **外部存储模式**：存储设置支持本地 / 外部 PostgreSQL 两种模式，外部模式保存时自动测试连接并初始化远端表
- **备份/恢复归位**：原本隐藏的数据导出备份能力移动到存储设置中，和存储模式统一管理

### 改进

- **存储文案整理**：存储配置入口标题改为“存储设置”，中文模式名调整为“本地 / 外部”，外部模式说明去除重复描述
- **连接表单调整**：手动填写连接信息时，账号输入框显示在数据库输入框前
- **远程模式流畅性**：列表读取复用连接配置并减少标签切换、搜索和统计刷新时的阻塞感
- **外部模式恢复刷新**：电脑从休眠或网络恢复后自动刷新列表和标签数量，避免需要手动切换标签才恢复显示
- **官网 SEO**：官网加入 Google Search Console 验证标记，并移除未支持平台说明
- **标题彩蛋文案**：版本号五击触发的标题调整为 `瑛楠的剪贴板`

### 技术细节

- **非商用许可**：新增 `LICENSE.md`，文档明确未经许可不允许商业用途
- **版本同步**：应用、Tauri 配置和 Rust 包版本同步为 `2.20.0`

- **官网**：新增 `docs/` 静态官网，支持中文/英文切换并可由 GitHub Pages 发布
- **官网版本号**：官网使用独立版本号 `1.0.1`，与软件版本号分开维护
- **官网系统支持**：官网新增系统支持区，标注 Windows x64 当前支持

### 改进

- **启动显示稳定性**：窗口启动时先定位再显示，避免左侧短暂白窗闪烁
- **托盘提示**：托盘图标悬浮提示显示当前语言下的软件名
- **快捷键唤起逻辑**：窗口被其他应用遮挡时按快捷键会重新拉到前台，只有已在前台时才隐藏
- **官网可访问性与 SEO**：增加键盘跳转入口、清晰焦点态、语言按钮状态、搜索关键词、社交分享信息和结构化数据
### 技术细节

- 备份/恢复代码暂时保留为隐藏能力，入口不在设置面板展示
- **版本同步**：应用、Tauri 配置和 Rust 包版本同步为 `2.15.1`

## v2.1.0 (2026-07-03)

### 修复

- **图片剪贴板去重**：图片历史现在使用真实像素数据参与 SHA-256 去重，不再只按尺寸和字节长度判断，避免同尺寸图片被误判为重复
- **搜索请求状态**：修复搜索防抖后可能用旧关键词查询、并触发重复请求的问题
- **查询边界**：剪贴板和备忘录查询的 `limit/offset` 增加范围修正并改为参数绑定，避免异常分页参数拖慢查询
- **回收站备忘录主题色**：回收站中的备忘录标题、正文和时间颜色改为主题变量，深色模式下保持一致
- **备忘录标签配色**：用户手动添加的备忘录标签跟随备忘录配色；自动识别标签继续沿用剪贴板分类颜色

### 改进

- **自动标签识别集中**：备忘录自动标签类型由后端分类器统一判断，前端只负责翻译和展示，减少剪贴板分类与备忘录标签逻辑漂移
- **组件拆分**：拆出回收站备忘录条目和确认弹窗组件，降低 `App.tsx` 复杂度
- **存储版本标记**：数据库初始化写入 `schema_version`，为后续显式迁移打基础
- **版本同步**：应用、Tauri 配置和 Rust 包版本同步为 `2.1.0`

## v2.0.2 (2026-07-01)

### 修复

- **检查更新结果判断**：修复后端返回 `has_update/latest_version/download_url`，前端读取 `hasUpdate/latestVersion/downloadUrl` 导致检查更新始终显示“已是最新版本”的问题
- **更新接口兼容性**：后端更新信息统一输出 camelCase，前端同时兼容 camelCase 与 snake_case 字段，避免后续字段名不一致再次误判

## v2.0.1 (2026-07-01)

### 变更

- **启动显示窗口**：软件启动后直接显示桌面主窗口，不再只在后台托盘常驻
- **多进程托盘修复**：包含单实例启动修复，反复点击桌面快捷图标或重复启动程序时会唤起已有窗口，不再创建多个进程和重复托盘图标
- **点击即粘贴现状记录**：文档补充当前自动粘贴依赖焦点回切和延迟发送 Ctrl+V，部分应用或慢焦点切换场景下可能不够灵敏，暂不在本版本重构
- **版本同步**：应用、Tauri 配置和 Rust 包版本同步为 `2.0.1`

## v2.0.0 (2026-07-01)

### 新功能

- **托盘设置重置位置**：右键托盘图标选择「设置」打开应用时，窗口位置自动重置为右半屏居中默认位置
- **应用正式命名**：软件名和项目名统一为 `SuperClipboard`，中文名为「超级剪贴板」
- **仓库名同步**：GitHub 仓库名同步为 `SuperClipboard`
- **首次语言跟随系统**：首次启动根据系统语言自动选择中文或英文，托盘菜单同步初始化
- **备忘录图片粘贴**：备忘录编辑区支持 Ctrl+V 粘贴图片，预览区自动渲染为图片，不再把 base64 文本暴露给用户
- **存储分开显示**：底栏存储按页面分别显示——备忘录页显示备忘录内容占用，剪贴板页显示剪贴板内容占用，SQL 排除已归档条目
- **默认初始状态同步**：默认快捷键为 `Alt+X`，主题模式跟随系统，主题配色为少年蓝，开机自启和自动检查更新默认开启，窗口置顶、原格式预览、备忘录和回收站默认关闭

### 改进

- **归档→回收站**：「归档模式」重命名为「回收站」，更符合用户直觉。归档按钮图标从文件夹（📁/📦）统一更换为废纸篓（🗑️），包括剪贴板条目和备忘录的删除按钮
- **底栏高度统一**：修复剪贴板页底栏因清除按钮 padding 导致的高度不一致，设置 min-height 并缩减按钮内边距
- **默认位置居中**：默认窗口位置从贴近右边缘改为右半屏水平居中（`(left + right*3)/4 - win_w/2`）
- **启动位置统一**：软件启动时先将隐藏主窗口移动到与托盘右键「设置」相同的默认位置
- **工作区感知**：窗口定位使用 `GetMonitorInfoW` 获取显示器工作区域（排除任务栏），确保窗口不会被任务栏遮挡
- **主线程安全**：快捷键回调中的窗口操作统一派发到主线程执行（`run_on_main_thread`），避免跨线程窗口操作导致的崩溃
- **原生窗口定位**：托盘设置入口使用 Windows `SetWindowPos` API 辅助定位，并在 `show()` 前后各调用一次确保位置生效
- **普通唤起保留位置**：快捷键和托盘左键唤起不再每次重置到默认位置，保留用户拖动后的当前窗口位置
- **改名数据迁移**：从 `SuperClipboard3` 升级到 `SuperClipboard` 时自动迁移旧数据目录中的 `clipboard.db`

### 修复

- **快捷键注册竞态**：修改 `set_shortcut` 为先注册新快捷键再注销旧快捷键，防止注册失败时丢失快捷键
- **快捷键录制**：支持 Windows 键组合，录制期间临时禁用全局唤醒/隐藏；再次点击录制按钮可取消，相同快捷键重新保存也会重新生效
- **窗口位置重置**：修复普通唤起窗口时每次被重置到默认位置的问题；现在只有托盘右键菜单「设置」会重置窗口位置
- **备忘录暗黑模式**：备忘录条目样式从硬编码颜色改为 CSS 变量，暗黑模式下正确适配
- **重复托盘图标**：增加单实例启动，反复点击快捷图标时唤起已有窗口，不再创建新的托盘实例

### 清理

- 移除未使用的 `storageSize` 翻译键和 `getCursorPosition` 前端 API
- 移除隐藏的保存位置/跟随模式运行时分支，`window_position` 模块仅保留设置入口默认定位与边界修正
- 新安装首次创建数据库时写入默认设置，已有用户升级时不覆盖已保存设置

### 技术细节

- 新增 `WindowPositionService::default_position()`：计算主屏右半区域中心并对工作区边界进行修正
- 新增 `set_window_position_native()` 辅助函数：封装 `SetWindowPos` 调用，设置入口 show 前后各定位一次
- Cargo.toml 保留 `Win32_Graphics_Gdi` feature（`GetMonitorInfoW`、`MonitorFromPoint` 所需）

## v2.0.0 Preview (2026-06-27)

### 新功能

- **主题三合一按钮**：将原来的「跟随系统」开关 + 浅色/深色按钮合并为一个三段式切换按钮（浅色 / 深色 / 自动），操作更直观
- **自定义备忘录颜色**：设置面板新增颜色选择器，支持 8 种预设色 + 自定义 HEX 输入，独立于主题色
- **备忘录归档**：备忘录支持软删除归档，归档 Tab 内分「剪贴板」和「备忘录」两个子标签，30 天自动清理
- **归档倒计时**：归档条目显示剩余天数（30 天自动过期），黄色徽章提示
- **跟随模式**：快捷键唤起窗口时自动定位到鼠标附近（托盘点击不触发），可在设置中开关
- **点击即粘贴**：快捷键唤起窗口后点击条目，自动隐藏窗口并模拟 Ctrl+V 粘贴到之前的活动窗口
- **备忘录拖拽排序**：使用 Pointer Events 重写拖拽（替代 HTML5 Drag-and-Drop），在 Tauri WebView2 中更稳定
- **智能新建按钮**：备忘录新建按钮支持切换行为——正在编辑空备忘录时再次点击关闭编辑器，有内容时保存并关闭
### 改进

- **分类算法升级**：
  - 新增 Markdown 内容检测（标题、加粗、链接、列表等）
  - 新增 YAML/TOML 配置文件检测
  - 新增正则表达式字面量检测
  - JSON 检测更严格：要求带引号的键名，避免误判
  - 分类阈值从 4 分提高到 5 分，减少误分类
  - 自然语言惩罚降低（-2 分），仅对单行文本生效
- **Stats 补全**：统计接口现在包含邮箱和文件路径类别计数
- **代码重构**：提取共享 TrashIcon 组件、去重 SQLite 行映射逻辑、统一全局快捷键回调
- **设置面板分组**：重新组织为「系统设置」和「功能设置」两个区域，视觉更清晰
- **条目背景固定**：所有条目（剪贴板 + 备忘录）使用固定灰白色 `#f5f5f5`，不随主题变化
- **备忘录文字可选**：备忘录内容区域支持文字选中复制（拖拽手柄和操作按钮仍不可选）
- **归档图标适配**：开启归档功能后，删除按钮自动切换为归档图标（📦/📁）
- **编辑徽章颜色统一**：备忘录「已编辑」徽章改用 memo-contrast 配色，与备忘录模块风格一致

### 修复

- **Hover 残留**：删除条目后，下方条目无需移出再移入即可正常触发 hover（改用 CSS `:hover` 替代 React state）
- **拖拽可靠性**：备忘录拖拽从 HTML5 Drag-and-Drop 改为 Pointer Events，解决 WebView2 中拖拽失灵的问题
- **备忘录列表乱跳**：修复切换条目时列表顺序异常的问题

### 清理

- 移除未使用的翻译键（`memoDrawerTitle`、`daysAgo`、`shortcutHint`）
- 移除未使用的 CSS 动画（`slideInRight`、`fadeIn`）
- 修复 i18n 上下文默认值与 `DEFAULT_LOCALE` 不一致的问题
- 移除前端未使用的 import 和死代码分支

## v1.5.0 (2026-06-23)

### 新功能

- **剪贴板内容编辑**：点击条目上的编辑按钮可修改剪贴板内容，编辑后显示「最后编辑时间」，原内容折叠保留，可通过下拉按钮展开查看
- **原格式预览开关**：设置面板新增「原格式预览」选项，开启后以等宽字体完整显示剪贴板原始内容
- **自动检查更新**：设置面板新增「自动检查更新」开关，开启后每次启动应用自动检查新版本，版本号显示在设置面板右上角

- **备忘录功能**：新增独立备忘录模块，与剪贴板分开存储（SQLite `memos` 表）
  - 支持标题、正文、标签
  - 支持搜索、置顶、删除
  - 每次按键实时保存到数据库，切换条目自动加载最新数据
  - 可在设置中开启/关闭备忘录入口
  - 备忘录标签位于"全部"左侧，使用紫色主题

- **窗口置顶开关**：设置面板新增"窗口置顶"选项，默认开启
  - 切换后立即生效，设置持久化到 SQLite

- **数据库大小显示**：底栏实时显示数据库文件占用空间

### 改进

- **默认快捷键改为 Shift+V**：更简短的唤起快捷键（可在设置中自定义）
- **标签栏滚轮滑动**：Tab 栏支持鼠标滚轮水平滚动，应对标签过多溢出的场景
- **剪贴板分类器重构**：
  - 从单一正则匹配改为评分制，多信号加权判断
  - 新增 CSS 检测：属性声明（`property: value;`）、选择器块（`.class { ... }`）
  - 新增 HTML/XML 标签检测
  - 新增 SQL 关键词检测（SELECT / INSERT / JOIN 等）
  - 新增 Shell 命令检测（`sudo`, `npm`, `docker`, `git` 等）
  - 扩充代码关键词覆盖：Rust、Go、Python、JS/TS、PHP、Java 等
  - 支持单行代码片段识别（不再要求必须多行）
  - 自然语言文本不会被误判为代码

### 修复

- **UTF-8 截断崩溃**：修复中文等多字节字符在生成预览时因字节截断导致 panic（改用 `chars().take(n)`）
- **备忘录切换数据不刷新**：点击其他备忘录条目后，编辑器始终从数据库重新加载最新数据，而非使用内存中的旧值
