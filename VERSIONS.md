# SuperClipboard 版本日志

## v3.5.0（2026-07-17）

剪贴板批量整理、分类更新与稳定性优化版本。

- 新增剪贴板多选模式，可从底栏进入并按当前列表顺序合并 2 至 20 个同主分类条目
- 多选模式支持混合选择不同分类并批量删除；开启回收站时原条目进入回收站，否则确认后永久删除
- 多选能力由实验功能中的“多选模式”开关控制，默认关闭；开启后可使用底栏入口、`Ctrl+点击` 和 `Delete` 快捷操作
- 统一经典与新版 UI 底栏字段、图标和按钮的高度及垂直中心，并校正带文字按钮中图标与中文字体的轻微上下偏移
- 底栏条目数与 Tab 数据统计统一，剪贴板、分类、备忘录和回收站不再只显示当前已加载的 50 条
- 合并前可选择仅生成合并结果，或在合并成功后同时删除原条目，并明确提示实际删除方式
- 文本、链接、邮箱、路径和代码条目合并后生成新的剪贴板记录；已有相同结果时复用现有记录，原条目不会被删除
- 多张图片合并后生成可继续编辑、插入文字和调整图片顺序的备忘录；备忘录功能关闭时会给出明确提示
- 本地 SQLite 与外部 PostgreSQL 使用统一批量读取接口，减少外部存储模式下的往返请求
- 增加合并规则、选择限制、内容顺序和图片备忘录生成的自动化测试
- 剪贴板多标签显示关闭时只按主标签生成分类列表和数量；开启后辅助标签才参与分类，一个混合内容条目可以出现在多个对应 Tab 中
- 修复新收录的 `E:/folder/file.ext` 形式 Windows 路径被识别为代码和文本的问题
- 实验功能新增“更新历史分类”按钮，建议在版本或分类规则更新后由用户主动执行；软件不会自动改写本地或外部存储中的旧数据
- 手动重新识别只更新主标签、辅助标签和搜索索引，跳过图片且不修改正文、时间、置顶或编辑记录
- 从分类标签清除历史时，可在弹窗中选择仅清除当前标签或清除全部未置顶记录；“全部”标签继续使用原有确认流程
- 清除历史弹窗会根据回收站是否开启，明确说明记录将进入回收站并可恢复，或被永久删除且无法恢复
- 回收站底栏新增“清空”按钮，只永久删除当前剪贴板或备忘录子标签中的全部记录，并在执行前显示数量与不可恢复提示
- 外部 PostgreSQL 的永久删除、批量删除、过期清理和回收站清空改为物理删除；普通清除历史不会再连带删除回收站中的记录
- 清空后会使仍在执行的旧列表和分页请求失效，避免远程响应延迟造成已删除条目短暂回填
- GitHub Releases API 被限流或临时不可用时，更新检查会自动通过公开 Latest Release 页面继续判断版本
- PostgreSQL schema 升级为 `9`，迁移会清理历史软删除记录、移除未被客户端读取的 `sync_events` 事件表，并按数据库记录已应用的分类规则版本；多设备刷新继续使用 `LISTEN / NOTIFY`
- 实验功能会显示当前分类规则版本及历史数据状态，切换本地或不同外部数据库时分别读取，不会混用状态
- 剪贴板与回收站备忘录的分页和旧请求控制拆分为独立 Hook；SQLite 与 PostgreSQL 的建表升级拆分到专用迁移模块
- 设置类面板和备忘录使用独立动态构建块，主入口 JavaScript 从约 `533 KB` 降至约 `244 KB`
- 新增 GitHub Actions CI，除前端与 Rust 全量检查外，还会启动临时 PostgreSQL 16 执行外部存储集成烟测
- 修复窗口置顶时需要按两次全局快捷键才能隐藏的问题；普通模式的前台恢复逻辑和最小化恢复行为保持不变
- 经典界面的剪贴板、备忘录和回收站改由列表背景统一形成分隔线，并移除可能触发 WebView2 合成层漏绘的条目绘制隔离
- 修复新版 UI 在大量条目同时加载时被 flex 布局压缩成密集横线的问题，剪贴板和备忘录条目现在始终保留实际内容高度
- 经典界面条目不足一屏时，列表剩余区域恢复为主背景色，分隔线只保留在实际条目之间
- 设置菜单的系统设置与功能设置分组统一为实验功能菜单的轻量标题与右侧细线样式
- 清理未使用的 Tauri Shell 插件与权限，并将回收站子标签重复样式收敛为统一 CSS；重新审计入口引用、样式、模块和依赖后未发现可安全删除的失联业务文件，完整平台图标集继续保留以便后续适配
- 应用、Tauri 配置、Rust 包、双语文档和官网软件版本同步为 `3.5.0`

## v3.4.0 (2026-07-16)

编辑内容去重与图片操作完善版本。

- 编辑过的剪贴板条目额外保存当前内容哈希；再次复制修改后的内容不会重复收录，SQLite 与外部 PostgreSQL 都会自动迁移历史数据
- 图片条目悬浮操作区新增应用内预览和 PNG 另存为；预览弹窗支持点击遮罩或按 `Esc` 关闭
- 自动更新提示中的下载按钮增加独立高对比度悬浮、按下和焦点状态
- 统一旧备忘录时间戳为 UTC，修复切换页面后“编辑于”显示回退的问题；空标题按界面语言显示
- 支持将文字、链接、图片和图片文件直接拖入主窗口收录；导入内容会写入系统剪贴板，并使用现有本地/外部存储、分类和去重机制，备忘录编辑区不受影响
- 未修改内容直接保存时不会写入重复条目或编辑时间；内容重新改回首次复制的原文后，会自动移除原文折叠区与编辑记录
- 编辑时间与“查看原文”统一放在条目右下角，当前内容与首次原文仍可分别点击复制
- 本地数据库 schema 升级为 `8`，外部 PostgreSQL schema 升级为 `6`
- 应用、Tauri 配置、Rust 包和双语文档版本同步为 `3.4.0`

## v3.3.0 (2026-07-15)

语言、存储切换与大数据量加载稳定性版本。

- 修复切换外部数据库后界面意外变成英文、语言按钮失效的问题；语言设置改为串行写入并支持失败回滚
- 本地与外部存储切换失败时恢复此前有效配置、表单和连接状态，并阻止重复提交
- 外部存储列表改为先加载轻量条目，图片进入可视区域时再按需读取，避免大批图片 Base64 导致列表超时或假空
- 剪贴板、备忘录和回收站使用每页 50 条增量加载，并补充本地 SQLite 与外部 PostgreSQL 排序索引
- 备忘录自动标签在保存时持久化，历史数据通过迁移回填，减少列表加载中的重复识别
- 设置修改按顺序持久化，更新检查增加请求超时、HTTP 状态校验、标准语义版本比较和 Windows 安装器直链
- 移除备份/恢复的测试版标签，保留跨版本恢复风险提示
- 经典 UI 的三个设置菜单恢复实体背景，新版 UI 将菜单移出标题栏合成层并修正生产 CSS 对背景滤镜的错误裁剪，避免两套界面样式相互影响
- 新版 UI 的三个设置菜单统一使用更清晰的专用边框色，增强标题分隔线、分区线及控件轮廓
- 拆分剪贴板、备忘录和更新检查后端模块，提取分页、排序、菜单事件 Hook，并新增 Vitest 前端测试基线
- 前端构建、ESLint、Vitest、Rust 格式检查、单元测试和 Clippy 检查均通过
- 应用、Tauri 配置、Rust 包、双语文档和官网软件版本展示同步为 `3.3.0`

## v3.2.0 (2026-07-13)

外部存储稳定性与多设备协作优化版本。

- 外部 PostgreSQL 使用受控连接池与后台任务，降低远程列表、搜索、计数和写入对界面流畅度的影响
- 接入 PostgreSQL `LISTEN / NOTIFY`，其他设备写入后当前设备会自动刷新条目与计数，无需切换标签
- 剪贴板和备忘录使用 `version` 乐观锁保护并发编辑；内容过期时拒绝覆盖并刷新最新数据
- 远端 schema 引入迁移记录和 advisory lock，保证首次初始化与后续升级串行执行
- 本地 / 外部存储调用收敛到统一存储边界，远端连接资料拆分为独立前端模块，减少重复状态逻辑
- 前端构建、ESLint、Rust 格式检查、单元测试和 Clippy 检查均通过
- 应用、Tauri 配置、Rust 包和双语文档版本同步为 `3.2.0`
